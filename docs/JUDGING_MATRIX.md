# Katiba OS judging matrix

| Criterion | What judges can see now | Why it matters |
|---|---|---|
| Legal and problem relevance | End-to-end small-claims evidence and review workflow | Targets a concrete access-to-justice bottleneck, not a generic Q&A use case |
| Innovation and creativity | Four engines on one provenance, identity, and audit core | Reframes legal AI as workflow infrastructure |
| Technical implementation | React PWA, Express API, role/object auth, normalized SQLite, PDF generation, tests | Demonstrates frontend, backend, data, AI, and integration boundaries |
| Data usage | Structured cases, evidence, timelines, issues, citations, and audit events | Makes legal facts queryable, explainable, and reviewable |
| AI integration | Schema-constrained analysis, evidence-ID filtering, safe fallback, human gate | Adds tangible organization without pretending to replace judgment |
| User experience | Separate citizen, paralegal, and lawyer surfaces; mobile and accessibility support | Serves non-lawyers and professionals with role-appropriate next actions |
| Feasibility and scalability | Compact local MVP plus PostgreSQL/object storage/worker evolution | Shows a credible path from demo to high-volume service |
| Potential impact | Legal-aid pilot plus SME cross-subsidy model | Connects civic impact with sustainable operations |

## Bonus considerations

| Bonus | Evidence in Katiba OS |
|---|---|
| Privacy, security, privilege | consent event, access checks, source integrity, audit log, trust center, production security plan |
| Accessibility and inclusion | plain-language copy, English/Kiswahili intake, focus states, responsive 375 px layout, reduced motion, PWA |
| Transparency and anti-corruption | attributable actions, source-backed findings, checksum visibility, human approvals |
| Community pilot potential | measurable clinic pilot and assisted-use workflow |
| Business viability | professional Contract/Compliance revenue can cross-subsidize Justice access |
